# Глобальные инструкции для Claude

## 🚨 ТЫ В СИСТЕМЕ CLAUDE CONTEXT KEEPER (командный режим)

Ты подключён к MCP-серверу `claude-logger`. Система сохраняет контекст работы всех разработчиков над проектами на этом сервере.

## 👤 ИДЕНТИФИКАЦИЯ РАЗРАБОТЧИКА

**Первое, что ты делаешь в сессии — определяешь, с кем работаешь:**

Спроси пользователя: «С кем я работаю? Назови своё имя для журнала (например: alice, bob, lead-dev)»

Сохрани это имя. Используй его во всех вызовах:
- `start_task(..., developer="alice")`
- `complete_task(..., developer="alice")`

Если пользователь не назвал имя — используй `developer="unknown"`.

---

## 🔍 ПЕРВЫЙ ШАГ С ПРОЕКТОМ

**Прежде чем работать с проектом, проверь его статус:**

1. Посмотри, есть ли в проекте директория `.claude/`
2. Если `.claude/` **отсутствует** → это старый проект, мигрируй (см. ниже)
3. Если `.claude/` **существует** → работай по стандартному流程у

---

## 📦 Стандартный рабочий процесс (проект уже в системе)

### В начале сессии:
1. Прочитай `.claude/claude.md` — там весь контекст, включая КТО последний работал
2. Вызови `restore_context(project_path="...")`
3. Если обнаружены прерванные задачи → покажи их пользователю
4. Спроси: «Продолжить прерванную задачу или начать новую?»

### Работа над задачей:
1. **ОДИН РАЗ в начале** → `start_task(project_path="...", task_name="...", developer="имя")`
2. После каждого значимого действия → `log(project_path="...", action_type="...", description="...")`
3. Важные решения → `log_decision(project_path="...", category="...", question="...", decision="...", reasoning="...")`
4. **В конце задачи** → `complete_task(project_path="...", summary="...", next_steps=[...])`

### При завершении сессии:
1. `complete_task(...)` — заверши текущую задачу
2. Скажи: «Чекпоинт создан. Следующий разработчик увидит состояние проекта.»

---

## 🏚️ Миграция старого проекта (нет .claude/)

1. Скажи: «Обнаружен проект без системы контекст-кипера. Мигрирую.»
2. Вызови `migrate_project(project_path="...")`
3. Прочитай сгенерированный `.claude/claude.md`
4. Вызови `restore_context(project_path="...")`
5. Работай по стандартному流程у

---

## 👥 КОМАНДНАЯ РАБОТА — ВАЖНЫЕ ПРАВИЛА

1. **Всегда указывай `developer`** — чтобы было видно, кто что сделал
2. **Читай claude.md перед работой** — там видно, кто последний работал и что сделал
3. **Создавай чекпоинт после каждой задачи** — чтобы коллега мог продолжить
4. **Если видишь незавершённую задачу другого разработчика:**
   - Скажи пользователю: «Задача от {developer} не завершена. Продолжить или начать новую?»
   - Если продолжаешь — укажи в description: «Продолжено после {предыдущий разработчик}»
5. **Не удаляй чужие чекпоинты** — они нужны для истории

---

## 🎯 Типы действий для log()

| Тип | Когда использовать |
|---|---|
| `file_created` | Создан новый файл |
| `file_modified` | Изменён существующий файл |
| `command_executed` | Выполнена консольная команда |
| `task_progress` | Промежуточный прогресс задачи |
| `issue_found` | Обнаружена проблема/баг |
| `issue_resolved` | Проблема решена |
| `handoff` | Передача задачи другому разработчику |

---

## 📋 Доступные инструменты MCP

| Инструмент | Ключевые параметры |
|---|---|
| `migrate_project` | `project_path` |
| `start_task` | `project_path`, `task_name`, `developer` |
| `log` | `project_path`, `action_type`, `description`, `details` |
| `log_decision` | `project_path`, `category`, `question`, `decision`, `reasoning` |
| `complete_task` | `project_path`, `summary`, `next_steps`, `developer` |
| `restore_context` | `project_path` |
| `regenerate_claude_md` | `project_path` |

---

## ⚡ КЛЮЧЕВЫЕ ПРИНЦИПЫ

- **Ты всегда в курсе контекста** благодаря чекпоинтам
- **Новый проект** → мигрируй и работай
- **Знакомый проект** → восстанови контекст и продолжай
- **Другой разработчик** → покажи, кто что сделал, спроси что делать
- **Не теряй контекст** → всегда создавай чекпоинт при завершении задачи
