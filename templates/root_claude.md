# Глобальные инструкции для Claude

## 🚨 ТЫ НАХОДИШЬСЯ В СИСТЕМЕ CLAUDE CONTEXT KEEPER

Ты подключён к MCP-серверу `claude-logger`. Он сохраняет контекст твоей работы между сессиями и аккаунтами.

## 🔍 ПЕРВЫЙ ШАГ — ВСЕГДА

**Прежде чем что-то делать с проектом, проверь его статус:**

1. Посмотри, есть ли в проекте директория `.claude/`
2. Если `.claude/` **отсутствует** → это старый проект, используй **Правила для старых проектов**
3. Если `.claude/` **существует** → это новый проект, используй **Правила для новых проектов**

---

## 📦 Правила для НОВЫХ проектов (есть .claude/)

### В начале сессии:
1. Прочитай `.claude/claude.md` — там весь контекст
2. Вызови `restore_context(project_path="...")`
3. Если обнаружены прерванные задачи — продолжи с них

### Работа над задачей:
1. **ОДИН РАЗ в начале** → `start_task(project_path="...", task_name="...")`
2. После каждого действия → `log(project_path="...", action_type="...", description="...")`
3. Важные решения → `log_decision(project_path="...", category="...", question="...", decision="...", reasoning="...")`
4. **В конце задачи** → `complete_task(project_path="...", summary="...", next_steps=[...])`

### Перед окончанием лимита токенов:
1. `complete_task(...)` — заверши текущую задачу
2. `export_prompt(project_path="...")` — получи промпт для нового аккаунта
3. Скажи пользователю скопировать промпт и вставить в новый чат

---

## 🏚️ Правила для СТАРЫХ проектов (нет .claude/)

### Сразу при обнаружении:
1. Скажи пользователю: «Обнаружен проект без системы контекст-кипера. Сейчас мигрирую.»
2. Вызови `migrate_project(project_path="...")`
3. Прочитай сгенерированный `.claude/claude.md`
4. Вызови `restore_context(project_path="...")`
5. Продолжи работу по **Правилам для новых проектов**

### Мигратор сам:
- Проанализирует структуру проекта
- Определит стек технологий
- Импортирует git-историю
- Создаст начальный чекпоинт
- Сгенерирует `.claude/claude.md` с описанием проекта

---

## 🎯 Типы действий для log()

| Тип | Когда использовать |
|---|---|
| `file_created` | Создан новый файл |
| `file_modified` | Изменён существующий файл |
| `command_executed` | Выполнена консольная команда |
| `task_progress` | Промежуточный прогресс задачи |
| `issue_found` | Обнаружена проблема/баг |
| `issue_resolved` | Проблема решена |

---

## 📋 Доступные инструменты MCP

| Инструмент | Назначение |
|---|---|
| `migrate_project` | Миграция старого проекта (один раз) |
| `start_task` | Начать новую задачу |
| `log` | Записать действие (буферизуется) |
| `log_decision` | Записать архитектурное решение |
| `complete_task` | Завершить задачу + создать чекпоинт |
| `restore_context` | Восстановить контекст в начале сессии |
| `export_prompt` | Экспортировать промпт для нового аккаунта |
| `regenerate_claude_md` | Перегенерировать claude.md |

---

## ⚡ КЛЮЧЕВОЕ ПРАВИЛО

**Ты всегда знаешь контекст проекта благодаря этой системе.**
- Новый проект → мигрируй и работай
- Знакомый проект → восстанови контекст и продолжай
- Меняется аккаунт → экспортируй промпт

Никогда не спрашивай пользователя «с чего начать» — ты уже знаешь из чекпоинтов.
