# Глобальные инструкции для Claude

## 🚨 ОБЯЗАТЕЛЬНЫЙ рабочий процесс

### В начале сессии с новым проектом:
1. Если проект создан до установки системы → `migrate_project(project_path="...")`
2. Прочитай `.claude/claude.md` проекта
3. Вызови `restore_context(project_path="...")`
4. Если есть прерванные задачи — продолжи с них

### В начале сессии с уже знакомым проектом:
1. Прочитай `.claude/claude.md`
2. Вызови `restore_context(project_path="...")`
3. Продолжи с последнего чекпоинта

### Работа над задачей:
1. **ОДИН РАЗ** → `start_task("название")`
2. По мере работы → `log(action_type, description)` (буферизуется)
3. Важные решения → `log_decision(category, question, decision, reasoning)`
4. **В конце задачи** → `complete_task(summary, next_steps)`

### Перед окончанием лимита:
1. Вызови `complete_task()` для текущей задачи
2. Вызови `export_prompt()` → скопируй результат
3. Вставь промпт в первый чат на новом аккаунте

## Типы действий для log()
- `file_created` — создан новый файл
- `file_modified` — изменён существующий
- `command_executed` — выполнена команда
- `task_progress` — промежуточный прогресс
- `issue_found` — обнаружена проблема
- `issue_resolved` — проблема решена

## Миграция существующих проектов
Если проект создан до установки Claude Context Keeper:
1. Вызови `migrate_project(project_path="...")` — один раз
2. Система проанализирует структуру, определит стек, импортирует git-историю
3. После миграции работай как обычно
